# Como contribuir

Obrigado por ajudar a tornar o Spark Mentor mais útil para quem está começando.

## Contribuições bem-vindas

- correções conceituais sobre Spark, PySpark ou Spark SQL;
- exemplos SQL/PySpark mais claros;
- novos exercícios introdutórios;
- melhorias de acessibilidade e responsividade;
- correções de português;
- relatos de problemas ao abrir o app em diferentes navegadores.

## Antes de enviar uma alteração

1. Mantenha o foco em estudantes iniciantes.
2. Explique o conceito em linguagem simples antes de aprofundar.
3. Quando possível, parta de um conceito SQL conhecido e mostre sua tradução para Spark SQL ou PySpark.
4. Preserve a arquitetura estática, o carregamento sob demanda e as versões fixadas do runtime.
5. Teste as áreas Estudo de hoje, Trilha, Laboratório Semântico, Primeiros passos, Tutor, Glossário e Progresso.
6. Não inclua chaves, tokens, dados pessoais ou datasets restritos.

Execute `npm test` antes de enviar o pull request. Alterações no núcleo Python também devem instalar `runtime/requirements-dev.txt` e passar em `npm run test:python`.

## Correções de conteúdo

Ao reportar uma possível imprecisão, informe:

- aula ou seção afetada;
- texto atual;
- correção sugerida;
- referência técnica, de preferência documentação oficial.

## Pull requests

Use um título direto e descreva:

- o problema resolvido;
- o que mudou;
- como a mudança foi validada;
- possíveis impactos no progresso salvo no navegador.

Ao contribuir, você concorda que sua alteração será disponibilizada sob a licença MIT do projeto.
